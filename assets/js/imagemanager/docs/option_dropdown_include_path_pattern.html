Option: dropdown.include_path_pattern

This is like an extra filer, you can filter the dropdown through this pattern to include folders from the dropdown list. Please be aware the you can get some rather strange behaviour regarding this option.

In short, this option is a regex match for the paths displayed in the dropdown selection. One example configuration could be if you blank the filesystem.directory_exclude_pattern and include pattern, and set the dropdown.exclude_path_pattern to /images/i, you will then only display paths on your website that has "images" in the path of the folder.

An example of website folder structure.

/wwwroot/
/wwwroot/about
/wwwroot/about/images
/wwwroot/contact
/wwwroot/contact/images
/wwwroot/projects
/wwwroot/projects/images

Example of configuration

$mcImageManagerConfig['filesystem.directory_exclude_pattern'] = "";
$mcImageManagerConfig['filesystem.directory_include_pattern'] = "";

$mcImageManagerConfig['dropdown.include_path_pattern'] = "/images/i";
$mcImageManagerConfig['dropdown.exclude_path_pattern'] = "";

The output in the dropdown would be.

/about/images
/contact/images
/projects/images

Example of usage in the config.php file:

$mcImageManagerConfig['dropdown.include_path_pattern'] = "/images/i";

Example of usage in a mc_access file:

dropdown.include_path_pattern=/images/i
